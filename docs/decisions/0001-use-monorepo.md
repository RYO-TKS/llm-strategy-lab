# ADR 0001: Use Monorepo Structure

## Status

Accepted

## Context

このプロジェクトは、データ取込、戦略、バックテスト、評価、LLM 補助、UI、API を段階的に追加していく前提です。
初期段階では責務ごとにコードを分けたい一方、リポジトリを分割するとセットアップと運用コストが先に増えます。

## Decision

初期段階ではモノレポを採用します。
ドメイン境界は `packages/` と `apps/` のディレクトリ構成で先に切り、必要になった段階で内部パッケージ化を進めます。

## Consequences

利点は、Issue 駆動での横断変更がしやすいこと、CI 設定を一箇所に集約できること、Phase 1 での手戻りを減らせることです。

欠点は、境界が曖昧だと肥大化しやすいことです。そのため、各ディレクトリの責務は `docs/` と README で都度明文化します。
