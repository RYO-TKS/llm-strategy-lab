# Architecture Overview

`llm-strategy-lab` は、研究用の速度と再現性の両立を狙ったモノレポ構成を採用します。

## レイヤー

`packages/` はドメインロジックを集約する層です。共通定義、データ取込、戦略、バックテスト、評価、LLM、サンドボックス、実験ランナーを分離します。

`apps/` は UI と API の公開面です。Phase 1 では実装よりも契約の明文化を優先し、先に README と配置を固定します。

`configs/` は実験条件を保持する層です。設定をコードから分離し、比較実験と再現実行をしやすくします。

`runs/` は生成物の出力先です。コード、設定、成果物を分離し、実験の痕跡を追いやすくします。

## 設計原則

再現可能性を優先します。研究用コードでも、設定と成果物の対応が追えることを前提にします。

拡張しやすさを優先します。最初は空実装でも、責務境界を先に切っておくことで後続実装の衝突を減らします。

運用ファイルを先に固定します。Issue template、PR template、CI を先に整備し、実装が増えても進め方がぶれない状態を作ります。
